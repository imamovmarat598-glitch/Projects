# Project Roadmap
## GPS Camera App

**Версия:** 1.0
**Дата:** 14 января 2026
**Автор:** Product & Engineering Team

---

## Обзор

Этот документ описывает план разработки GPS Camera App от MVP до полнофункционального продукта с расширенными возможностями. Разработка разделена на 3 основных фазы.

---

## Phase 1: MVP (Minimum Viable Product)
**Срок:** 2-3 месяца
**Цель:** Запустить базовую версию для валидации концепции и получения первых пользователей

### Месяц 1: Основа и камера

#### Неделя 1-2: Настройка проекта
- [ ] Инициализация React Native проекта
- [ ] Настройка TypeScript конфигурации
- [ ] Подключение основных библиотек (Navigation, Redux, etc.)
- [ ] Настройка CI/CD pipeline (GitHub Actions / Bitrise)
- [ ] Настройка линтеров и форматтеров (ESLint, Prettier)
- [ ] Создание базовой структуры папок
- [ ] Настройка темы и стилей (Material Design / HIG)

**Deliverable:** Работающий скелет приложения с навигацией

#### Неделя 3-4: Модуль камеры
- [ ] Интеграция react-native-vision-camera
- [ ] Реализация Camera Screen UI
- [ ] Обработка разрешений на камеру
- [ ] Захват фото и сохранение в файловую систему
- [ ] Создание миниатюр для галереи
- [ ] Базовые настройки камеры (вспышка, переключение)
- [ ] Unit тесты для CameraService

**Deliverable:** Рабочая камера с сохранением фото

### Месяц 2: GPS и хранение данных

#### Неделя 5-6: GPS модуль
- [ ] Интеграция react-native-geolocation-service
- [ ] Реализация GPSService
- [ ] Обработка разрешений на геолокацию
- [ ] Отображение GPS данных в реальном времени на Camera Screen
- [ ] Индикатор качества GPS сигнала
- [ ] Запись GPS координат в EXIF метаданные
- [ ] Обработка edge cases (нет GPS, слабый сигнал)
- [ ] Unit тесты для GPSService

**Deliverable:** Фото с GPS метаданными в EXIF

#### Неделя 7-8: База данных и хранение
- [ ] Настройка SQLite (react-native-sqlite-storage)
- [ ] Создание схемы БД (photos, tags, photo_tags, settings)
- [ ] Реализация PhotoStorage сервиса
- [ ] CRUD операции для фотографий
- [ ] Миграции БД
- [ ] Индексирование для оптимизации запросов
- [ ] Интеграция Redux для управления состоянием
- [ ] Unit тесты для PhotoStorage

**Deliverable:** Работающее хранилище с БД

### Месяц 3: Галерея и карта

#### Неделя 9-10: Gallery Screen
- [ ] Реализация Gallery Screen UI
- [ ] Grid layout для фотографий
- [ ] Lazy loading и виртуализация списка
- [ ] Photo Detail Screen с метаданными
- [ ] Редактирование заметок и категорий
- [ ] Добавление и управление метками (tags)
- [ ] Фильтрация по датам и категориям
- [ ] Поиск по заметкам
- [ ] Удаление фотографий (soft delete)

**Deliverable:** Полнофункциональная галерея

#### Неделя 11-12: Map Screen
- [ ] Интеграция react-native-maps
- [ ] Реализация Map Screen UI
- [ ] Отображение маркеров фотографий на карте
- [ ] Кластеризация маркеров (> 50 точек)
- [ ] Bottom Sheet с превью фото при клике на маркер
- [ ] Переключение типов карт (Standard/Satellite/Hybrid)
- [ ] Кнопка "Моя локация"
- [ ] Фильтры для карты
- [ ] Навигация к Photo Detail с карты

**Deliverable:** Интерактивная карта с фотографиями

#### Неделя 13: Финализация MVP
- [ ] Settings Screen с базовыми настройками
- [ ] Экспорт в CSV формат
- [ ] Onboarding flow для новых пользователей
- [ ] Обработка всех edge cases и ошибок
- [ ] Оптимизация производительности
- [ ] Багфиксинг
- [ ] Integration тесты для основных flows
- [ ] Beta тестирование (TestFlight / Internal Testing)

### Релиз MVP v1.0.0
- [ ] Подготовка материалов для сторов (скриншоты, описания)
- [ ] Публикация в App Store
- [ ] Публикация в Google Play
- [ ] Мониторинг отзывов и crash reports
- [ ] Сбор первой пользовательской обратной связи

**KPI для MVP:**
- ✅ 1,000+ установок за первый месяц
- ✅ 4.0+ рейтинг в сторах

## Phase 2: Enhancement & Growth
**Срок:** 2-3 месяца после MVP
**Цель:** Добавить функции на основе обратной связи, улучшить UX, начать монетизацию

### Месяц 4: Расширенный экспорт и организация

#### Неделя 14-15: Экспорт в разные форматы
- [ ] Реализация ExportService
- [ ] Экспорт в KML формат (Google Earth)
- [ ] Экспорт в GPX формат (GPS навигация)
- [ ] Экспорт в ZIP с фото и метаданными
- [ ] UI для выбора формата и опций экспорта
- [ ] Share API интеграция
- [ ] Batch экспорт (множественный выбор)
- [ ] Прогресс индикатор для экспорта

**Deliverable:** Полнофункциональный экспорт данных

#### Неделя 16-17: Расширенная организация
- [ ] Пользовательские категории (не только предустановленные)
- [ ] Цветовое кодирование меток
- [ ] Папки/альбомы для группировки фото
- [ ] Массовые операции (изменение категории, добавление меток)
- [ ] Продвинутые фильтры (по GPS точности, по размеру файла)
- [ ] Сортировка (по дате, имени, расстоянию от текущей точки)
- [ ] Избранное (Favorites)

**Deliverable:** Продвинутые инструменты организации

### Месяц 5: Наложение данных и Premium

#### Неделя 18-19: Наложение данных на фото

- [ ] Создание шаблонов наложения (угол, полоса, центр)

## Риски и зависимости

### Технические риски
| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Проблемы с GPS точностью | Средняя | Высокое | Тщательное тестирование, настройки точности |
| Производительность на слабых устройствах | Средняя | Среднее | Оптимизация, минимальные требования |
| Сложность синхронизации | Высокая | Высокое | Тщательное проектирование, тестирование |
| AR совместимость | Средняя | Среднее | Graceful degradation, fallback |

### Бизнес риски
| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Низкая конверсия в Premium | Средняя | Высокое | A/B тестирование, улучшение value proposition |
| Конкуренция | Высокая | Среднее | Уникальные функции, отличный UX |
| Проблемы с масштабированием | Низкая | Высокое | Правильная архитектура с начала |

### Зависимости
- Картографические API (Google Maps / Mapbox)
- App Store / Google Play approval процессы
- Третьесторонние библиотеки React Native
- Cloud провайдеры (AWS / GCP)
